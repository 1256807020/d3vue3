## Bug 修复: 模板市场导入无效

- **根因**: DemoHub 模板市场"打开编排器查看"按钮只导航 `/pipeline`，未传 `?tpl=xxx` 参数
- **修复**: 保存 `lastImportedId`，导航时传 `?tpl=${lastImportedId}`
- **附加**: PipelineEditor 加载模板后自动调用 `fitToContent()` + `updateZoomPercent()`

## Bug 修复: 画布空白区无法拖拽平移

- **根因**: 平移 mousedown 挂在 `<svg>` 元素上，用 `event.target !== svgRef.value` 判断空白区。
  但 grid 层的 `<rect>` 覆盖整个虚拟画布，鼠标点击空白实际命中 grid rect，永远不满足条件
- **修复**: 平移 mousedown 改挂到 `canvasRef.value` 容器 div，用 `target.closest('.node/.connection/.port')` 排除节点元素
  - 节点拖拽、端口连线等内部 handler 的 stopPropagation 保证了不会误触发平移

## 技术评估结论

- 推荐技术栈：`@dagrejs/dagre`（图布局）+ `d3.js`（渲染/交互动画）+ `Vue3`（状态/路由/组件）
- `@vue-flow/core` 适用于快速标准流程编辑器，不适合深度定制场景（本项目已有深度 d3 定制）
- `bpmn.js` 太重（2MB+），仅适用于需要 BPMN 2.0 标准 XML 输出的场景
- 项目已有 FlowEngine 基类（状态机/事件/审批等待/history），可在此基础上扩展规则引擎（json-rules-engine）、表单引擎（Formily）、消息引擎（WebSocket）

## 流程执行引擎重构 — 状态机模式

- **移除** `computeExecutionOrder()` 拓扑排序（无法处理循环/branching）
- **替换为**状态机驱动的执行：
  - `getOutgoingEdges(nodeId)` — 获取所有出边（按 action 优先级排序）
  - `findNextNode(nodeId, action)` — action 路由（精确匹配或默认第一条）
  - `executeCurrentNode()` — 执行当前活跃节点，完成后找后继
  - `animateTokenToCurrent()` — 动画从 prevNode 沿连线飞到 activeNode
- **支持分支**: 一个节点多条出线时，按 action 标签（通过/驳回/转审）路由
- **支持循环**: 不再依赖 DAG 假设，通过 maxSteps=50 防止无限循环
- **状态变量**: `activeNodeId` / `prevNodeId` / `activeAction` / `executionHistory` 替代旧的 `executionOrder` / `executionStepIndex`

## PipelineEditor 视图优化

- **工具栏紧凑化**: height 50→38px, gap 8→2px, padding/字体缩小
- **Figma 风格平滑缩放**: `smoothViewBox()` 使用 `d3.interpolate` + `d3.easeCubicOut` 做 viewBox 平滑过渡
- 滚轮缩放防抖(30ms) + 以视图中心为基准
- `fitToContent()` 和 `resetView()` 使用平滑动画过渡(duration 350-400ms)
- **修复**: `smoothViewBox` 调用前不应预更新 panX/viewW，否则插值起点=终点无动画 → 改为只传目标值
- **修复**: `executeEndNode()` 中 `prevNodeId=null` 清除了动画路径，导致最后两步无连线动画 → 移除该行
- **修复**: `smoothViewBox` 从 SVG viewBox 属性读实际值（而非 JS 变量中间态），中断旧动画再启新动画 → 缩放居中
- **缩放锚点修复**: 从"视图中心"改为"内容区中心"(`getContentCenter()`)，确保缩放后节点始终在屏幕中央
- **导入 JSON 优化**: 从 `prompt()` 原生弹窗 → Element Plus Dialog + textarea + 格式校验 + 数组验证
  - 导入后自动调 `fitToContent()` 居中
- **默认流程也居中**: `initDefaultNodes()` 后自动 `fitToContent()`
- **多循环支持**: `executedActions` 是 per-node 的 Map，多个不同审批节点各自独立追踪驳回/通过路径，互不干扰
- **Bug 修复**: `saveLayout()` 直接用 `localStorage.setItem('vqs_flow_templates',...)` 与 FlowEngine 的 `'flow-templates-v3'` key 不一致 → 改为 `FlowEngine.saveTemplate(tpl)`
- **README.md**: 生成企业级文档，含技术架构、核心算法（正交路径/端口优化/状态机执行/Figma缩放/网格吸附）、特色功能、Bug 修复记录、硬编码清单、Nest.js 改造方案、Prisma Schema、经验总结、路线图
  - 兜底 fallback: 无精确 action 匹配时，优先走"通过"边，不再盲目走第一条边
- **审批流模板更新**: 6 节点 (新增"重新填写表单") + 6 条带 action 标签连线
